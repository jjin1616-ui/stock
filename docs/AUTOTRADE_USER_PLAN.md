# KoreaStockDash 자동매매 사용자 관점 기획안 (v1)

작성일: 2026-02-16

## 1) 사용자 목표
- "추천 종목을 내가 직접 계속 누르지 않아도 규칙대로 자동으로 거래하고 싶다."
- "실제 돈이 나가더라도 왜 거래됐는지, 얼마나 벌고 잃었는지 한눈에 보고 싶다."
- "사고가 날까 불안하니, 실주문 전에 모의로 충분히 검증하고 싶다."

## 2) 핵심 사용자
- 초보 사용자
  - 자동매매 경험이 없고, 실주문 사고를 가장 두려워함
  - 필요한 것: paper 기본값, 쉬운 설명, 강한 안전장치
- 중급 사용자
  - 단타/급등/관심을 같이 보고 판단하는 사용자
  - 필요한 것: 소스별 ON/OFF, 예산/주문수/손실한도 제어
- 고급 사용자
  - 실주문을 원하는 사용자
  - 필요한 것: broker 상태, 히스토리, 성과 추적, 빠른 롤백

## 3) 사용자 여정
1. 하단 `자동` 탭 진입
2. 설정 확인 (`paper`, 소스 포함 여부, 예산/주문수/손실한도)
3. `모의 점검` 실행
4. 후보/예상 주문 결과 확인
5. 문제 없으면 `실행`
6. 히스토리/수익 지표 확인
7. 필요 시 즉시 OFF

## 4) 화면 정보 구조
- 상단: 자동매매 상태(ON/OFF), 마지막 실행 시간
- 설정 카드:
  - 실행 환경(`paper/demo/prod`)
  - 소스 선택(단타/급등/논문/장투/관심)
  - 종목당 예산, 1회 주문 수, 일 손실한도
- 실행 카드:
  - `모의 점검`
  - `실행`
- 후보 요약 카드:
  - 후보 수, 상위 티커
- 성과 요약 카드:
  - ROI, 승률, MDD, 미실현 손익
- 주문 히스토리 리스트:
  - 소스, 티커, 상태, 사유, 손익

## 5) 사용자 신뢰 원칙
- 기본값은 무조건 `paper`
- 실주문은 명시적 조건(환경+활성화+서버 허용) 없으면 불가
- 상태 숨김 금지:
  - `PAPER_FILLED`
  - `BROKER_SUBMITTED`
  - `BROKER_REJECTED`
  - `SKIPPED`
- "왜 스킵/거절됐는지" reason 필수 노출

## 6) 실패/예외 UX
- KIS 설정 누락:
  - 메시지: `증권 연동 정보가 설정되지 않아 실주문을 실행할 수 없습니다.`
- 일 손실한도 초과:
  - 메시지: `일 손실 한도 초과로 자동주문이 중지되었습니다.`
- 가격/수량 계산 불가:
  - 메시지: `가격 또는 수량 조건이 맞지 않아 주문이 건너뛰어졌습니다.`
- 네트워크 실패:
  - 메시지: `통신 오류로 실행 결과가 지연되고 있습니다. 잠시 후 다시 시도하세요.`

## 7) 사용자별 증권 계정 입력 UX (필수)
- 앱 `자동매매` 탭에서 사용자별 KIS 정보를 직접 입력/저장 가능
- `사용자별 계정 사용` 스위치로 `내 계정` vs `서버 공용 계정` 선택 가능
- 입력 항목:
  - 앱키/시크릿
  - 계좌번호
  - 계좌상품코드
  - 모의/실전은 실행 환경 선택(`demo/prod`)으로 분리
- 보안 원칙:
  - 앱은 평문 저장 금지
  - 서버 저장 시 사용자별 암호화 + 마스킹 표시
  - 저장된 키 원문은 재조회 불가(입력 시 업데이트)

## 8) 릴리즈 수용 기준 (사용자 관점)
- 초보 사용자가 3분 안에 `paper 모의 점검`을 성공할 수 있어야 한다.
- 사용자 1명이 자신의 계정으로만 실행되고, 다른 사용자 계정과 절대 섞이면 안 된다.
- 실행 후 10초 이내에 히스토리 카드에서 상태/사유/손익이 보여야 한다.
- 손실한도 초과 시 추가 주문이 즉시 중단되고 명확한 경고가 떠야 한다.

## 9) 다음 단계 우선순위
1. 실주문 전 `테스트 연결` + `paper 대비 확인` 단계 강제
2. 체결동기화(접수/체결/미체결) 실시간 반영
3. 청산 로직 및 실현손익 고도화

# 가설 검증 원페이지 명세서 (v1.0)
기준일: 2026-02-11 (KST)  
목적: 현재 서비스 중인 `단타/장투/논문/급등/미장` 가설(알고리즘)의 **실증 유효성**을 제3의 AI가 동일 기준으로 검증하도록 표준화한다.

## 1) 역할 정의
너는 퀀트 리서치 검증 엔진이다. 목표는 “좋아 보이는 설명”이 아니라 “실제 성과/안정성/운영 가능성” 검증이다.  
확인되지 않은 수치 생성 금지. 불확실하면 `미확인`으로 표기.

## 2) 검증 대상 가설
1. 단타: `TA/RE/RS` 결합 점수 + 게이트(`gate_on`) 기반 진입 가설.  
2. 장투: 유동성(`value_ma20`) + 20거래일 추세(`ret20>0`) + 보수적 진입구간 가설.  
3. 논문: 현재 단타 엔진 재사용 구조가 독립 전략으로 유효한지.  
4. 급등: 실시간 등락/거래대금/거래량/비율/인기 모드가 단기 후행수익 예측력 있는지.  
5. 미장: SEC Form4 `CEO/CFO + Non-derivative + Code=P` 신호의 유효성 및 데이터 안정성.

## 3) 필수 검증 질문
1. 각 전략은 OOS(Out-of-Sample)에서 양(+)의 기대값을 유지하는가?  
2. 성과가 특정 구간(상승장/테마장)에만 편중되는가?  
3. 거래비용/슬리피지 반영 후에도 우위가 남는가?  
4. 데이터 결손/지연(특히 SEC 403/429) 시 운영 품질이 허용 가능한가?  
5. 논문 탭은 단타와 실제로 차별적 알파를 제공하는가?

## 4) 데이터/분할 기준
1. 검증 기간: 최근 24개월 이상.  
2. 분할: `학습(튜닝 금지) 0 / 검증 12개월 / OOS 12개월` 원칙.  
3. 레짐 분해: 상승/하락/횡보, 고변동/저변동 구간별 성과 분리.  
4. 해외(미장): 제출일 필터 후 거래일 재판정 원칙 유지.

## 5) 공통 측정 지표
1. 수익성: 누적수익, CAGR, 평균 R, 기대값(Expectancy).  
2. 안정성: MDD, 변동성, 샤프/소르티노, 월별 승률.  
3. 실행성: 신호당 체결 가능성, 회전율, 평균 보유기간.  
4. 운영 품질: API 실패율, fallback 비중, 지연시간(p50/p95).  
5. 편향 점검: 종목/섹터/시총 쏠림, 소수 종목 기여도.

## 6) 합격/실패 기준 (초안)
1. OOS 기대값 > 0, OOS MDD가 기준 한도 이내.  
2. 비용 반영 후도 기대값 양수 유지.  
3. 레짐 3개 중 2개 이상에서 성과 붕괴 없음.  
4. 운영 품질: 데이터 실패/지연이 사용자 체감 임계 초과 시 실패.  
5. 논문 탭: 단타 복제 수준이면 “독립 가설 실패”로 판정.

## 7) 산출물 형식 (반드시)
1. 전략별 결과표 1개: `단타/장투/논문/급등/미장` 행, 핵심 지표 열.  
2. 전략별 판정: `통과/조건부/실패` + 한 줄 근거.  
3. 우선순위 개선안 Top 5: 영향도(High/Med/Low) + 구현 난이도.  
4. 재현 정보: 데이터 범위, 파라미터, 커밋/버전, 실행 시각(KST).

## 8) 즉시 개선 권고 기준
1. 단타: 게이트 임계값 적응형(quantile) vs 고정값 A/B.  
2. 장투: 추세 필터 단일(`ret20`)에서 다중(60/120일) 확장 검증.  
3. 급등: 신호 발생 후 10분/30분/종가 수익률 추적 KPI 추가.  
4. 미장: SEC 차단 시 캐시 재사용 품질 점수와 신선도 표기 의무화.  
5. 논문: 단타 재사용 종료, 독립 피처/목표함수 분리 전까지 “실험” 라벨 유지.
